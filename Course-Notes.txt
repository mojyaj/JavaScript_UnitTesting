					it('should summarize all number values in an array')

			'Test' 
				Provide an annonymous function
				example,
					it( '...' , () => {
						// test code
					})

	'expect()' function
		Define expectations or success criteria
		Arguments
			The stored 'results' returned from 'arranging' the test
		
	'.to' chain functions *See Assertion Library
		Examples,
			expect(result).toBe(expect))
			expect(result).toBeFalse(expect))
			expect(result).toBeTrue(expect))
			expect(result).toBeLessThan(expect))

'package.json'
	Placement: "scripts:"
		"test:" 
			--reporter verbose  flag
				Provides a  more detailed  test output
				"scripts": {
					"vitest --run --reporter verbose", ...
									^^^^^^^^^^^^^^^
					"test:watch": "vitest",
					"start": "http-server -c-1"
				}
			--run
				Only execute test file once and exit
				Without '--run',
					Vitest enters watch mode by default
					On save a file, tests re-run automatically
					Enter 'Watch mode' - gives an interactive terminal UI with live feedback

	Command: 'npm run test:watch'
		will be the same as running command: 'vitest'
		Enter 'Watch mode and tests re-run automatically on saved changes

20. The AAA Pattern - Arrange, Act, Assert

3 Acts
	Arrange - Define the testing environment & values
	Act - Run the actual code /function that should be tested
	Assert - Evaluate the produced value/result and compare it to the expected value/result

24. Testing For Errors

Testing Errors in functions
	1) Declare a variable (function reference)
	2) Wrap test function inside function expression, arrow function, or normal function or reference
	3) Pass variable to 'expect()' and chain with '.toThrow()'
		example, 
			expect(var).toThrow()

Tips:	.not
	This chain funciton tests for the opposite
	example,
		*expect(var).not.toThrow() // test that 'var' did not throw error

26. Testing For Thrown Errors & Error Messages

Testing for Specific Error Messages
	- 'toThrow()'
		Checks that a function throws an error, and optionally verifies what kind of error it is.
		*can take 1 optional argument
		Method Definition:
			(method) Vi.JestAssertion<() => void>.toThrow(expected?: string | RegExp | (new (...args: any[]) => any) | Error | undefined): void
			Breakdown:
				- thing inside expect() must be a function reference
				- expected?, "?" means optional
					- option1  string	('Expected Error Message')
						real failed out example,	'TypeError: numbers is not iterable'
					- option2  RegExp	(Match error message with a pattern)
					- option3  Error	(Check exact error object instance)
					- option4  Error Constructor (A class or constructor function)
					- option5  undefined	(not passed anything)
	
Tips:
	- Narrow down the range of errors that could be thrown to expect the right kind of error
